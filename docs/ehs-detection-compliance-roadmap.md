# EHS 检测报告合规模块排期与实施方案

> 本文用于评估并拆解“企业检测点检测报告 / 现场采样数据自动合规判定”方向的工程量。
> 核心原则：数值型合规判断不依赖大模型裁决，而是通过结构化数据、限值库、单位换算、pandas 计算和确定性规则完成；大模型只作为报告解析、字段归一、解释文本生成的辅助能力。

## 1. 目标范围

### 1.1 业务目标

系统后续支持企业上传检测报告或现场采样数据，自动完成：

1. 解析检测点、检测因子、检测值、单位、采样时长、检测日期、岗位/排口等信息。
2. 根据法规、标准或企业适用标准中的限值，自动判断是否超标。
3. 对需要计算的指标进行计算，例如 TWA / PC-TWA、STEL、日均值、等效声级等。
4. 输出明确结论：合格、超标、临界、数据不足、需人工复核。
5. 对超标项给出违反依据，包括标准编号、标准名称、条款/表号、限值、单位、适用条件。

### 1.2 不建议一开始覆盖的范围

以下能力工程量较大，建议后置：

1. 全类型 PDF / 扫描件 / 图片报告 100% 自动识别。
2. 全国、地方、行业所有 EHS 标准和历史版本完整覆盖。
3. 自动判断复杂标准适用优先级，例如行业标准、地方标准、综合排放标准、特别排放限值之间的优先级。
4. 自动从国家法规网站实时同步并解析所有标准。
5. 完全无人审核的法律合规结论。

## 2. 技术路线

### 2.1 总体链路

```text
检测报告 / 现场采样数据
        ↓
文件解析 / 表格导入
        ↓
人工确认或自动结构化
        ↓
单位归一与数据校验
        ↓
pandas 计算指标
        ↓
匹配法规限值库
        ↓
确定性规则判定
        ↓
结果入库 + 前端展示 + 违反依据
```

### 2.2 大模型的边界

适合使用大模型的部分：

1. PDF / DOCX 报告中字段抽取的辅助。
2. 检测因子别名归一，例如“苯 / Benzene / CAS 71-43-2”。
3. 报告格式识别和表格语义理解。
4. 生成说明性文字、整改建议、复核提示。
5. 标记数据缺失，例如缺少采样时长、单位不明确、岗位班次缺失。

不适合使用大模型的部分：

1. 判断 `检测值 > 限值`。
2. 计算 TWA / STEL / 日均值 / 超标倍数。
3. 决定最终合规状态。
4. 编造标准编号、条款和限值。

### 2.3 结论原则

数值型合规结果应由确定性规则给出：

```python
if calculated_value > limit_value:
    status = "EXCEEDED"
else:
    status = "COMPLIANT"
```

每个结论必须带来源：

```json
{
  "indicator": "苯",
  "sample_point": "喷漆岗位",
  "calculated_value": 8.6,
  "unit": "mg/m3",
  "limit_type": "PC-TWA",
  "limit_value": 6,
  "standard_code": "GBZ 2.1-2019",
  "standard_name": "工作场所有害因素职业接触限值 第1部分：化学有害因素",
  "clause": "表1",
  "compliance_status": "EXCEEDED",
  "exceedance_multiple": 1.43
}
```

## 3. 三层建设计划

## 第一层：结构化数据 MVP

### 3.1 阶段目标

先不追求复杂 PDF 自动解析，优先支持 Excel / CSV / 手工录入数据，跑通确定性合规判定闭环：

```text
上传 Excel/CSV → 结构化字段校验 → pandas 计算 → 匹配限值 → 判断超标 → 展示违反依据
```

### 3.2 建议首批指标范围

建议先选一条业务线做深，不要同时覆盖职业卫生、废水、废气、噪声、危废全部内容。

优先方案 A：职业卫生

1. 苯、甲苯、二甲苯。
2. 粉尘。
3. 噪声。
4. 高温 WBGT（按 GBZ 2.2-2007 表8 的劳动强度 / 接触时间率拆成结构化限值）。
4. 可计算 TWA / PC-TWA / STEL 的基础样本。

优先方案 B：环保废水

1. pH。
2. COD。
3. 氨氮。
4. 总磷。
5. 总氮。

### 3.4 后端模块建议

建议新增模块：

```text
app/services/detection_import_service.py
app/services/detection_calculation_service.py
app/services/detection_limit_service.py
app/services/detection_compliance_service.py
app/dao/detection_dao.py
app/schemas/detection_schema.py
app/api/v1/endpoints/detection.py
```

职责拆分：

1. `detection_import_service.py`
   负责 Excel / CSV 解析、字段映射、基础校验。

2. `detection_calculation_service.py`
   负责 TWA、STEL、平均值、超标倍数等计算。

3. `detection_limit_service.py`
   负责按检测因子、介质、报告类型、企业行业、地区等匹配限值。

4. `detection_compliance_service.py`
   负责输出最终合规判定。

新增或扩展以下页面：

1. 检测报告列表页。

3. 结构化数据预览页。
4. 合规判定结果页。
5. 限值库维护页。

结果页重点展示：

1. 超标项数量。

4. 违反标准编号、标准名称、条款/表号。
5. 数据不足或需复核项。

按一个人全栈开发估算：

| 天数 | 工作内容 | 交付物 |

| D2 | Alembic 迁移、DAO、schema、基础 API | 数据库表、CRUD 接口 |
| D3 | Excel / CSV 导入，字段校验，错误提示 | 可上传结构化检测数据 |
| D4 | 单位归一、基础计算逻辑、pandas 管线 | 计算服务和单元测试 |
| D5 | 限值库匹配、超标判定、违反依据输出 | 合规结果生成 |
| D6 | 前端列表、上传、结果详情页 | 可演示页面 |
| D7 | 限值维护页面、样例数据、端到端测试 | MVP 可验收版本 |

时间估算：

2. 如果首批只做 Excel/CSV + 少量指标：4-6 个工作日可见效果。
3. 如果同时做职业卫生和环保废水：建议按 8-12 个工作日估算。

## 第二层：检测报告解析自动化

### 4.1 阶段目标

在第一层结构化计算闭环跑通后，再支持 PDF / DOCX / 扫描 PDF 的自动解析。

目标链路：

```text
上传报告 → 自动抽取表格/文本 → 结构化候选结果 → 人工确认 → 进入计算判定
```

### 4.2 解析技术选型

建议按文件类型拆：

1. 数字 PDF
   - PyMuPDF
   - pdfplumber
   - Docling

2. 扫描 PDF / 图片
   - PaddleOCR
   - Docling OCR
   - 必要时接入商业 OCR

3. DOCX
   - python-docx
   - Unstructured

4. Excel
   - pandas / openpyxl

### 4.3 人工确认流程

报告解析不建议直接进入最终判定，应增加人工确认层：

1. 展示自动抽取出的检测点、检测因子、检测值、单位。
2. 标记低置信度字段。
3. 允许用户修改字段。
4. 保存确认后的结构化数据。
5. 再运行确定性计算和合规判定。

### 4.4 第二层排期

| 周期 | 工作内容 | 交付物 |
|---:|---|---|
| W1 | 收集 10-20 份脱敏报告；分类报告模板；实现 PDF/DOCX 基础解析 | 报告样本集、解析原型 |
| W2 | 表格抽取、字段映射、单位识别、置信度标记 | 自动结构化候选结果 |
| W3 | 人工确认页面、字段修正、重新计算 | 人工校正闭环 |
| W4 | 扫描件 OCR、异常报告兜底、测试集回归 | 可内测解析模块 |

时间估算：

1. 保守估算：2-4 周。
2. 如果报告模板高度统一：1.5-2 周。
3. 如果扫描件多、表格复杂、模板差异大：4 周以上。

## 第三层：法规限值库治理

### 5.1 阶段目标

将“限值表”从简单维护表升级为可长期治理的法规/标准限值库。

需要解决：

1. 标准版本。
2. 生效日期和废止日期。
3. 行业适用范围。
4. 地区适用范围。
5. 标准优先级。
6. 同一检测因子不同口径限值。
7. 来源文件和条款可追溯。
8. 缺口管理。

### 5.2 标准适用复杂度

例如环保排放场景可能同时存在：

1. 国家综合排放标准。
2. 行业排放标准。
3. 地方排放标准。
4. 排污许可证要求。
5. 企业内部控制指标。

最终适用限值并不总是“查到一个就用”，可能需要规则：

1. 地方标准严于国家标准时优先。
2. 行业标准优于综合标准。
3. 排污许可证可能有更严要求。
4. 特别排放限值适用于特定区域或特定行业。

这部分不能只靠大模型，需要产品规则和人工确认。

### 5.3 治理能力建议

建议逐步建设：

1. 限值来源文件管理。
2. 标准版本管理。
3. 限值人工录入和审核。
4. 适用条件配置。
5. 冲突限值提示。
6. 缺口主题记录。
7. 废止/替代关系维护。
8. 批量导入限值。

### 5.4 第三层排期

| 周期 | 工作内容 | 交付物 |
|---:|---|---|
| W1 | 限值库字段扩展、来源文件、版本、生效日期 | 可追溯限值库 |
| W2 | 适用条件和优先级规则配置 | 初版适用规则引擎 |
| W3 | 限值维护后台、审核流程、批量导入 | 维护型后台 |
| W4 | 缺口管理、冲突提示、回归测试 | 治理闭环 |
| W5-W8 | 扩展更多行业、地区、报告类型 | 可持续扩展版本 |

时间估算：

1. 小范围职业卫生或废水限值：4-6 周。
2. 职业卫生 + 环保主要指标：6-8 周。
3. 全 EHS 法规标准治理：3-6 个月起步，并且需要持续维护。

## 6. 总体排期建议

### 6.1 推荐实施节奏

| 阶段 | 范围 | 时间 | 价值 |
|---|---|---:|---|
| 第一层 MVP | Excel/CSV、少量指标、限值库、确定性超标判定 | 5-8 工作日 | 最快形成可演示能力 |
| 第二层报告解析 | PDF/DOCX/OCR、人工确认、解析回归 | 2-4 周 | 降低人工录入成本 |
| 第三层限值治理 | 版本、适用范围、优先级、废止替代、缺口管理 | 4-8 周起 | 支撑长期产品化 |

### 6.2 最小可验收版本

建议第一版只承诺：

1. 支持 Excel / CSV。
2. 支持一个报告类型，例如职业卫生。
3. 支持 5-10 个检测因子。
4. 支持限值库人工维护。
5. 支持明确输出超标项和违反依据。
6. 支持结果导出或前端查看。

第一版不承诺：

1. 任意 PDF 全自动识别。
2. 自动覆盖所有法规标准。
3. 自动判断所有行业/地区适用标准。
4. 无人工复核的法律结论。

## 7. 关键风险

### 7.1 报告样式不统一

不同检测机构的报告格式差异大，PDF 表格抽取不稳定。应通过“自动解析 + 人工确认”降低风险。

### 7.2 限值库不完整

限值库不完整会导致“无法判定”或“需复核”。系统需要区分：

1. 未超标。
2. 超标。
3. 未找到适用限值。
4. 数据不足。

不能把“未找到限值”当成“合规”。

### 7.3 标准适用规则复杂

行业、地区、排口类型、企业许可证可能影响适用限值。复杂规则应通过可配置方式逐步建设，不建议第一版写死所有情况。

### 7.4 单位换算风险

单位换算错误会直接导致误判。应建立单位换算白名单和测试用例。

### 7.5 OCR 准确率风险

扫描 PDF 依赖 OCR，识别错误需要人工确认。超标判断不能直接基于未确认的 OCR 数值。

## 8. 测试策略

### 8.1 单元测试

覆盖：

1. 单位换算。
2. TWA / STEL / 均值计算。
3. pH 范围判断。
4. 超标倍数计算。
5. 限值匹配。
6. 数据不足判断。

### 8.2 集成测试

覆盖：

1. 上传 Excel / CSV。
2. 解析结构化数据。
3. 运行计算任务。
4. 生成合规结果。
5. 前端查询结果。

### 8.3 样本回归

建立脱敏报告样本集：

1. 合格样本。
2. 超标样本。
3. 单位不同的样本。
4. 缺少采样时长的样本。
5. PDF 表格复杂样本。
6. 扫描 PDF 样本。

## 9. 人力和时间评估

按一个人全栈开发估算：

| 版本 | 范围 | 时间 |
|---|---|---:|
| P0 原型 | 手工录入/CSV、少量限值、后端判定、简单结果 JSON | 2-3 工作日 |
| P1 MVP | Excel/CSV、限值库、pandas 计算、前端结果页、测试 | 5-8 工作日 |
| P2 内测版 | PDF/DOCX 解析、人工确认、更多指标、回归样本 | 3-5 周 |
| P3 产品版 | 限值治理、适用规则、版本管理、缺口管理 | 2-3 个月 |

如果使用更强模型辅助开发，代码生成会快一些，但法规确认、样本测试、OCR 验证仍然是主要瓶颈。总体可以缩短 20%-30%，很难数量级缩短。

## 10. 建议下一步

### 10.1 需要先确定的问题

1. 第一批做职业卫生还是环保废水。
2. 第一批检测因子清单。
3. 是否已有固定 Excel 模板。
4. 是否已有脱敏检测报告样本。
5. 限值库第一版由谁确认。
6. 前端是否需要人工校正页面放在第一期。

### 10.2 推荐启动方案

建议先做职业卫生结构化 MVP：

1. 指标：苯、甲苯、二甲苯、粉尘、噪声、高温 WBGT。
2. 输入：Excel / CSV。
3. 输出：检测点维度合规结果。
4. 规则：PC-TWA / PC-STEL / MAC / 噪声基础限值 / 高温 WBGT 标量限值。
5. 依据：标准编号、标准名称、表号、限值。

这样可以在一周左右形成可演示闭环，再决定是否投入 PDF/DOCX/OCR 解析。
