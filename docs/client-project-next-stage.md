# 客户/项目机构版下一阶段候选

当前客户/项目 MVP 只交付轻量上下文字段：

- `assessment_tasks.client_name / project_name / project_code / service_type`
- `detection_reports.client_name / project_name / project_code / service_type`

独立客户公司表和项目/委托单表暂不进入当前阶段，避免在没有前端页面、业务关联和交付流程时暴露半成品接口。

`0011_client_company_project_tables` 保留为兼容用 no-op 迁移，只用于让已经记录过该 revision 的环境继续被 Alembic 识别；当前 MVP 不再创建或注册独立客户/项目表。

## 触发条件

- 第三方检测机构试点确认需要长期维护委托客户主数据。
- 同一委托单位下存在多项目、多报告、多评价任务归档诉求。
- 需要项目状态、复核人、签发、交付或客户确认流程。

## 候选范围

- 客户公司表：名称、统一社会信用代码、行业、地址、联系人、备注。
- 项目/委托单表：客户公司、项目名称、项目编号、服务类型、状态、起止日期、联系人、备注。
- 评价任务和检测报告关联项目，而不是只保存自由文本客户/项目字段。
- 项目汇总页：风险项、超标项、失败任务、待判定报告和最近动态。
- 组织权限保持不变：普通用户只读写本组织数据，管理员可查看全局。

## 不在当前 MVP 暴露

- `/api/v1/client-companies`
- `/api/v1/client-projects`
- 客户公司/项目管理前端页面
- 项目复核、签发、报告导出、客户确认
