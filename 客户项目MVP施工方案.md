# 客户/项目 MVP 施工方案

## 背景

当前系统已经具备评价任务、检测合规、工作台、组织权限、AI 合规助手等基础能力。现阶段如果继续扩展 Agent，会遇到一个核心问题：Agent 只能读“任务/报告”，还不知道这些数据属于哪个委托单位、哪个项目。

为了同时支撑企业和第三方检测机构，先补一层轻量客户/项目上下文，再让 Agent 读取这些字段。

## 产品定位

面向企业和第三方检测机构的 EHS 合规评价分析与检测判定平台。

企业使用目标：

- 汇总内部 EHS 合规状态。
- 上传检查材料、检测报告。
- 查看风险项、超标项、整改建议。
- 跟踪失败任务、待判定报告和整改闭环。

第三方检测机构使用目标：

- 按委托单位和项目归档评价/检测资料。
- 快速解析报告、匹配限值、形成初步分析。
- 复核风险项、检测判定和依据。
- 为后续报告交付、复核签发留出扩展空间。

## 本阶段范围

本阶段做“第三方友好 MVP”，不做完整机构版大重构。

新增轻量字段：

- `client_name`：委托单位 / 客户公司。
- `project_name`：项目名称。
- `project_code`：项目编号，可选。
- `service_type`：服务类型，可选，例如评价、检测、整改、综合。

涉及对象：

- 评价任务 `assessment_tasks`。
- 检测报告 `detection_reports`。

## 暂不做的内容

- 不新增完整客户公司表。
- 不新增项目/委托单独立表。
- 不重做权限模型。
- 不做复核签发流程。
- 不做报告导出闭环。
- 不让首页商务咨询接入业务 Agent。

这些留到机构版第二阶段。

## 施工顺序

1. 数据库和模型
   - 在评价任务、检测报告上增加客户/项目字段。
   - 增加 Alembic migration。
   - ORM 模型同步字段。

2. 后端接口
   - 评价任务创建/上传支持客户/项目字段。
   - 检测报告上传/导入支持客户/项目字段。
   - 列表接口支持按 `client_name`、`project_name`、`service_type` 筛选。
   - 响应 schema 返回客户/项目字段。

3. 前端页面
   - 评价任务上传表单增加客户/项目字段。
   - 评价任务列表增加客户/项目展示和筛选。
   - 检测报告上传/文档解析入口增加客户/项目字段。
   - 检测报告列表增加客户/项目展示和筛选。

4. 工作台
   - 待处理事项展示客户/项目上下文。
   - 最近任务、最近报告显示客户/项目。

5. Agent 只读适配
   - 工具返回任务/报告时带客户/项目字段。
   - 失败任务、待判定报告摘要中展示客户/项目。
   - 不扩展复杂推理能力，只做上下文补齐。

6. 首页和方案咨询
   - 首页文案明确“企业 + 第三方检测机构”均可使用。
   - 方案咨询入口保持 B2B 风格，强调试点评估、样例验证、私有化部署。

## 验收标准

- 评价任务上传后能保存并展示客户/项目字段。
- 检测报告上传后能保存并展示客户/项目字段。
- 评价任务列表可以按客户/项目筛选。
- 检测报告列表可以按客户/项目筛选。
- 工作台待处理事项可以看到客户/项目上下文。
- Agent 回答失败任务、待判定报告时包含客户/项目，不暴露内部错误字段。
- 普通用户仍只能看到自己组织范围内数据。
- 管理员仍能看到全局数据。

## 风险和注意事项

- 当前数据库可能已有历史任务/报告，新字段必须允许为空。
- 不要破坏现有上传和解析流程。
- 不要让第三方检测机构场景绕过组织权限。
- Agent 只读取字段，不做写入、不做重跑、不做交付结论承诺。
- 首页商务咨询不要变成 To C 客服，也不要接业务数据。

## 下一阶段候选

如果第三方检测机构场景验证成立，再升级完整机构版：

- 客户公司表。
- 项目/委托单表。
- 项目成员和复核人。
- 项目状态：待资料、分析中、待复核、已交付。
- 报告导出。
- 复核签发。
- 客户确认。
